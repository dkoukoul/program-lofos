# program.lofos.gr — Σκοπός & Λειτουργικό Πλαίσιο

**4ο Σύστημα Αεροπροσκόπων Ηρακλείου**

> Αυτό το έγγραφο περιγράφει ΜΟΝΟ τον σκοπό, τους χρήστες και τη λειτουργικότητα του site.
> Δεν περιέχει τεχνικές αποφάσεις (stack, βάση δεδομένων, hosting κ.λπ.) — αυτές θα οριστούν σε επόμενο στάδιο.

## 1. Σκοπός

Το πρόβλημα που λύνουμε σήμερα: το μηνιαίο πρόγραμμα δράσεων του Συστήματος οργανώνεται και επικοινωνείται με τρόπο διάσπαρτο και όχι φιλικό σε γονείς/παιδιά (π.χ. μηνύματα, χαρτί, ανεπίσημα αρχεία).

Ο στόχος του program.lofos.gr είναι διπλός:

1. **Εσωτερικό εργαλείο προγραμματισμού** — να επιτρέπει στους βαθμοφόρους κάθε τμήματος να ορίζουν εύκολα το πρόγραμμα δράσεων του τμήματός τους, και στο επιτελείο του Συστήματος να ορίζει δράσεις που αφορούν όλο το Σύστημα.
2. **Δημόσια, φιλική προβολή** — όταν ένα πρόγραμμα δηλωθεί "ολοκληρωμένο", να εμφανίζεται online, χωρίς login, σε μορφή απλή και ευχάριστη για γονείς και παιδιά, ανά τμήμα, με δυνατότητα συνδρομής μέσω iCal.

## 2. Γλωσσάρι

| Όρος | Σημασία |
|---|---|
| **Σύστημα** | Το 4ο Σύστημα Αεροπροσκόπων Ηρακλείου συνολικά. |
| **Τμήμα** | Μία από τις τρεις ηλικιακές βαθμίδες: **Αγέλη**, **Ομάδα**, **Κοινότητα**. Το Σύστημα έχει ακριβώς ένα τμήμα από κάθε τύπο. |
| **Βαθμοφόρος τμήματος** | Ενήλικας υπεύθυνος ενός τμήματος. Έως 4 ανά τμήμα, όλοι με ίδια δικαιώματα επεξεργασίας του προγράμματος του τμήματός τους. |
| **Επιτελείο Συστήματος** | Αρχηγός Συστήματος και Υπαρχηγοί Συστήματος. Μπορούν να ορίζουν δράσεις Συστήματος (αφορούν όλα τα τμήματα) και να επεμβαίνουν σε δράσεις οποιουδήποτε τμήματος. |
| **Δράση** | Μία προγραμματισμένη εκδήλωση, ένας από τέσσερις τύπους: **Τυπική** (καθιερωμένη Κυριακάτικη συγκέντρωση), **Ημερήσια** (εκδρομή/δράση εντός ημέρας εκτός τυπικού ωραρίου), **Διήμερη** (διήμερη/πολυήμερη δράση), **Άλλη** (οτιδήποτε άλλο, π.χ. συγκέντρωση σε διαφορετική μέρα/ώρα). |
| **Χωρίς δράση** | Ρητή σήμανση μιας ημερομηνίας (συνήθως Κυριακή) ως "καμία δράση δεν θα γίνει". Μοντελοποιείται ως ειδικός τύπος δράσης (καταλαμβάνει τη θέση, ώστε να μην προταθεί ξανά ως default και να μπλοκάρει/προειδοποιεί για νέα δράση την ίδια ημέρα) και εμφανίζεται ρητά στη δημόσια σελίδα, ώστε ο γονέας να ξέρει ότι δεν παραλείφθηκε κατά λάθος. |
| **Δράση Συστήματος** | Δράση που ορίζεται από το επιτελείο και αφορά ταυτόχρονα όλα τα τμήματα. Εμφανίζεται στο πρόγραμμα όλων των τμημάτων για την ίδια ημέρα/ώρα και αντικαθιστά τυχόν δράση τμήματος εκείνη τη στιγμή. |
| **Περίοδος προγράμματος** | Το χρονικό διάστημα που καλύπτει ένα "πρόγραμμα" προς δημοσίευση. Συνήθως ένας ημερολογιακός μήνας, αλλά μπορεί να είναι οποιοδήποτε διάστημα (π.χ. 15 Νοεμβρίου – 15 Δεκεμβρίου). Ορίζεται ελεύθερα από όποιον το δημιουργεί. |
| **Πρόχειρο (Draft)** | Κατάσταση προγράμματος όσο δουλεύεται από τους βαθμοφόρους — ορατό μόνο σε βαθμοφόρους/επιτελείο, όχι δημόσια. |
| **Δημοσιευμένο (Published)** | Κατάσταση προγράμματος αφού δηλωθεί "ολοκληρωμένο" — γίνεται ορατό δημόσια στη φιλική σελίδα του τμήματος. |
| **Επισκέπτης** | Γονέας ή παιδί που βλέπει το δημόσιο πρόγραμμα, χωρίς λογαριασμό. |

## 3. Χρήστες & Ρόλοι

### 3.1 Βαθμοφόρος τμήματος
- Ανήκει σε ένα από τα τρία τμήματα (Αγέλη / Ομάδα / Κοινότητα).
- Λογαριασμός δημιουργείται χειροκίνητα από τον admin (δεν υπάρχει self-registration).
- Μπορεί να δημιουργεί, επεξεργάζεται και δημοσιεύει το πρόγραμμα **μόνο του δικού του τμήματος**.
- Όλοι οι βαθμοφόροι ενός τμήματος έχουν ισότιμα δικαιώματα — δεν υπάρχει ιεραρχία μέσα στο τμήμα.
- Δεν χρειάζεται έγκριση από κανέναν άλλον για να δημοσιεύσει.

### 3.2 Επιτελείο Συστήματος (Αρχηγός / Υπαρχηγοί)
- Μπορεί να δημιουργεί **δράσεις Συστήματος**, που εμφανίζονται αυτόματα σε όλα τα τμήματα.
- Μπορεί να δει και να επεξεργαστεί το πρόγραμμα **οποιουδήποτε** τμήματος (ίδια δικαιώματα με τους βαθμοφόρους του τμήματος, plus).
- Είναι αυτός που έχει και τον ρόλο του admin: δημιουργεί λογαριασμούς βαθμοφόρων.

### 3.3 Επισκέπτης (γονέας / παιδί)
- Καμία σύνδεση.
- Βλέπει μόνο δημοσιευμένα προγράμματα, ανά τμήμα.
- Βλέπει κατά προτεραιότητα τον τρέχοντα/ενεργό μήνα, με πρόσβαση και σε ιστορικό παλαιότερων περιόδων.
- Μπορεί να εγγραφεί σε iCal feed ανά τμήμα.

## 4. Δομή δεδομένων (εννοιολογικά)

```
Σύστημα
 └── Τμήμα (Αγέλη / Ομάδα / Κοινότητα)
      └── Πρόγραμμα (περίοδος: π.χ. μήνας ή custom εύρος ημερομηνιών)
           └── Δράση
                - τύπος (Τυπική / Ημερήσια / Διήμερη / Άλλη / Χωρίς δράση)
                - τόπος
                - ώρα έναρξης, ώρα λήξης (ή ημερομηνία λήξης για Διήμερη)
                - συμμετέχοντες βαθμοφόροι
                - κόστος συμμετοχής (προαιρετικό)
                - τι να κρατάνε τα παιδιά (προαιρετικό)
                - επιπλέον δυναμικά πεδία (τίτλος + περιγραφή, όσα χρειαστούν, προαιρετικά)
                - flag "Δράση Συστήματος" (ναι/όχι)
```

Defaults ανά τύπο (επιταχύνουν τη συνηθισμένη περίπτωση — κάθε πεδίο παραμένει ελεύθερα επεξεργάσιμο):
- **Τυπική**: Κυριακή, ώρα έναρξης 11:00, ώρα λήξης 13:00, τόπος "Λόφος", τι να κρατάνε "παγούρι".
- **Άλλη**: τι να κρατάνε "πλήρης προσκοπική στολή" — τα υπόλοιπα πεδία χωρίς προεπιλογή.
- **Ημερήσια** / **Διήμερη**: defaults θα οριστούν αργότερα (§8).
- **Χωρίς δράση**: κανένα άλλο πεδίο δεν συμπληρώνεται — απλώς μαρκάρει την ημερομηνία.

## 5. Βασικές λειτουργίες

### 5.1 Δημιουργία / επεξεργασία προγράμματος (βαθμοφόρος τμήματος)
- Ορίζει περίοδο (π.χ. "Οκτώβριος 2026" ή custom εύρος).
- Προσθέτει δράσεις μία-μία: πρώτα επιλέγει ημερομηνία (default: η επόμενη διαθέσιμη Κυριακή εντός της περιόδου, δηλ. η πρώτη Κυριακή χωρίς ήδη καταχωρημένη δράση ή "Χωρίς δράση"), μετά τύπο (Τυπική/Ημερήσια/Διήμερη/Άλλη/Χωρίς δράση). Τα defaults ανά τύπο (§4) καλύπτουν τη συνηθισμένη περίπτωση με λίγα κλικ.
- Μπορεί να δηλώσει ρητά μια ημερομηνία ως "Χωρίς δράση", ώστε να φαίνεται ξεκάθαρα στους επισκέπτες ότι δεν θα γίνει τίποτα εκείνη τη μέρα (αντί απλώς να μην υπάρχει καταχώρηση).
- Το πρόγραμμα παραμένει σε κατάσταση "Πρόχειρο" μέχρι να το δηλώσει "Ολοκληρωμένο".
- Κατά την επιλογή ημερομηνίας για νέα δράση, ελέγχεται αν υπάρχει ήδη δράση (ίδιου τμήματος ή Δράση Συστήματος) εκείνη την ημέρα και εμφανίζεται προειδοποίηση. Η ακριβής συμπεριφορά (αποκλεισμός της αποθήκευσης ή προειδοποίηση με δυνατότητα συνέχειας) ρυθμίζεται μέσω environment variable — **default: δεν επιτρέπεται επικάλυψη** — βλ. [architecture-and-tech-stack.md §6](./architecture-and-tech-stack.md).

### 5.2 Δράσεις Συστήματος (επιτελείο)
- Δημιουργούνται μία φορά και προβάλλονται αυτόματα σε **όλα** τα τμήματα, χωρίς να χρειάζεται να τις προσθέσει ξεχωριστά κάθε βαθμοφόρος.
- Κατά τη δημιουργία δράσης, το επιτελείο έχει επιπλέον επιλογή/flag "Δράση Συστήματος" (μη ορατό/επιλέξιμο για βαθμοφόρους τμήματος).
- Το επιτελείο μπορεί επίσης να επεμβαίνει απευθείας σε πρόγραμμα συγκεκριμένου τμήματος όποτε χρειαστεί.

### 5.3 Δημοσίευση ("Ολοκληρωμένο")
- Ο βαθμοφόρος του τμήματος (ή το επιτελείο) δηλώνει το πρόγραμμα "Ολοκληρωμένο" όποτε είναι έτοιμο — χωρίς να χρειάζεται έγκριση τρίτου.
- Μόλις δημοσιευτεί, γίνεται αυτόματα ορατό στη δημόσια σελίδα του τμήματος.

### 5.4 Αλλαγές σε ήδη δημοσιευμένο πρόγραμμα
- Επιτρέπονται οποτεδήποτε (π.χ. αλλαγή ώρας λόγω καιρού).
- Κάθε πεδίο που άλλαξε μετά τη δημοσίευση (ώρα, τόπος, κ.λπ.) πρέπει να επισημαίνεται οπτικά στη δημόσια σελίδα, ώστε γονέας/παιδί να καταλαβαίνει αμέσως τι άλλαξε.
- Η αλλαγή ενημερώνει αυτόματα και το iCal feed.

### 5.5 Ειδοποιήσεις email
- Αποστέλλονται **μόνο** σε: βαθμοφόρους του συγκεκριμένου τμήματος + επιτελείο Συστήματος (Αρχηγό/Υπαρχηγούς).
- Trigger: (α) δημοσίευση νέου προγράμματος, (β) αλλαγή σε ήδη δημοσιευμένο πρόγραμμα.
- Δεν στέλνονται ειδοποιήσεις σε γονείς/παιδιά στη Φάση 1 (βλέπουν μέσω site/iCal).

### 5.6 Δημόσια σελίδα ανά τμήμα
- Προβάλλει κατά προτεραιότητα την τρέχουσα/ενεργή περίοδο προγράμματος.
- Παρέχει πρόσβαση και σε ιστορικό παλαιότερων δημοσιευμένων περιόδων.
- Καμία απαίτηση login.
- Κάθε τμήμα έχει δικό του οπτικό template, προσαρμοσμένο στην ηλικιακή του ομάδα (παιχνιδιάρικο/μύθος-θέμα για Αγέλη, περιπετειώδες/προσκοπικό για Ομάδα, μοντέρνο/απλό για Κοινότητα) — βλ. [docs/ux-ui-guidelines.md](./ux-ui-guidelines.md) για πλήρεις κατευθυντήριες γραμμές.
- Επιπλέον, ανά περίοδο (π.χ. μήνας) μπορεί να οριστεί προαιρετικό "θέμα" (τίτλος + εικόνα/mascot), κυρίως για την Αγέλη (π.χ. "Ο Μόγλης", "Minions"), αλλά τεχνικά διαθέσιμο σε όλα τα τμήματα.
- Ημερομηνίες που έχουν δηλωθεί ρητά ως "Χωρίς δράση" εμφανίζονται με σαφή, διακριτική ένδειξη (π.χ. "Δεν θα γίνει συγκέντρωση αυτή την Κυριακή") — ώστε να μην συγχέεται με απουσία δημοσιευμένου προγράμματος.

### 5.7 iCal feed
- Ένα μοναδικό, δημόσιο, μη μαντεύσιμο URL ανά τμήμα.
- Αντανακλά μόνο δημοσιευμένες δράσεις (ό,τι φαίνεται στη δημόσια σελίδα).
- Ενημερώνεται αυτόματα όταν γίνονται αλλαγές σε δημοσιευμένο πρόγραμμα.

## 6. Ροή κατάστασης προγράμματος

```
[Πρόχειρο] --(βαθμοφόρος/επιτελείο δηλώνει "Ολοκληρωμένο")--> [Δημοσιευμένο]
                                                                     |
                                                    (νέα αλλαγή σε πεδίο)
                                                                     v
                                              [Δημοσιευμένο, με ένδειξη "άλλαξε το Χ"]
```

## 7. Εκτός πεδίου Φάσης 1 (πιθανή Φάση 2)

- Ειδοποιήσεις προς γονείς/παιδιά (email ή push).
- Δήλωση συμμετοχής/παρουσιολόγιο από γονείς.
- Ενδιάμεσοι ρόλοι μέσα σε τμήμα (π.χ. αρχηγός κλάδου vs βαθμοφόρος).
- Αγγλική εκδοχή.
- Ενσωμάτωση/σύνδεση με το lofos.gr (θα παραμείνει ανεξάρτητο site).

## 8. Ανοιχτά σημεία προς διευκρίνιση αργότερα

- Ακριβής μορφή/παραδείγματα των "δυναμικών πεδίων" ανά δράση (§4) — ελεύθερο κείμενο, checkbox, κ.λπ.
- Ακριβή defaults (ώρα, τόπος, τι να κρατάνε) για τους τύπους Ημερήσια και Διήμερη — θα οριστούν αργότερα.
- Πόσο πίσω πηγαίνει το ιστορικό δημόσιας προβολής (χωρίς όριο, ή π.χ. 12 μήνες);
- Τι συμβαίνει αν το επιτελείο διαγράψει/αλλάξει δράση Συστήματος αφού βαθμοφόρος τμήματος έχει ήδη προγραμματίσει κάτι γύρω της.
